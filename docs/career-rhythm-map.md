# 커리어 검증 점수·소득리듬 맵 명세

## 한 문장 정의

커리어 검증 점수는 검증된 일을 이식 가능한 평판으로 보여주고, 소득리듬 맵은 달력 대신
긱워커의 실제 소득 사건에 맞춰 성장과 혜택을 보여준다.

## 층 분리

| 층 | 움직이는 입력 | 바꾸는 출력 | 절대 바꾸지 않는 것 |
|---|---|---|---|
| 검증 신뢰 | 검증 이력, GitHub·홈택스·KOSA 등 외부 자료 | 커리어 검증 점수, 잠정·준검증·확정 단계, 심사자료 연결 여부 | 상품 자격·금리·한도, 입금 배분 금액 |
| 소득 습관 | 고유 입금 거래에 연결된 배분 승인·조정 | 소득리듬 횟수, 맵 발판, 표시용 성장·혜택 | 커리어 검증 점수, 검증 단계, 심사자료 연결 여부 |

## 검증 단계

| 단계 | 결정 조건 | 제품에서 여는 것 |
|---|---|---|
| 잠정 | 외부 연결 0~1곳 | 검증 이력과 준비 안내 |
| 준검증 | 서로 다른 외부 연결 2곳 이상 | 다중 출처 확인 상태 표시 |
| 확정 | 홈택스 신고소득 또는 KOSA 경력 확인 | 연결된 검증자료를 상품 심사 화면으로 가져가는 통로 |

확정은 대출 승인, 신청 자격, 우대금리, 한도 증가를 의미하지 않는다. 최종 상품 조건은 은행의
실제 심사에서 결정한다.

## 맵 전진 규칙

- 첫 발판은 기존 검증 이력으로 시작한다.
- 외부 커리어 소스 1곳 연결은 `검증 신뢰` 발판 1개로 표시한다.
- `txn_id`가 있는 실제 입금 배분을 승인 또는 조정하면 `소득리듬` 발판 1개로 표시한다.
- 같은 입금 ID는 한 번만 센다.
- 거절, 목표 페이싱, 앱 열기, 단순 조회, 입금 ID 없는 수동 배분 제안은 전진시키지 않는다.
- 달력 기준 연속 출석을 쓰지 않으며, 일이 없는 달에도 이전 진행도를 잃지 않는다.
- 맵 진행과 캐릭터는 표시·혜택 영역이며 점수나 금융 판단의 입력이 아니다.

## 5단계 혜택 표

| 단계 | 표시용 성장 | 혜택 예시 |
|---|---|---|
| 1 | 성장 시작 | 기본 저금통 열림 |
| 2 | 첫 발판 | 하나머니 혜택 확인 |
| 3 | 리듬 형성 | 우대금리 쿠폰 확인 |
| 4 | 리듬 유지 | 파킹통장 우대 확인 |
| 5 | 한 여정 완성 | 환율·수수료 혜택 확인 |

혜택명은 기존 하나금융 프로모션 도구를 재타게팅하는 기획 예시다. 실제 지급 조건·대상·기간은
상품 및 프로모션 정책 확인 뒤 노출한다. 비상금대출 심사자료 연결은 이 표의 습관 보상이 아니라
검증 신뢰 층의 `확정` 단계 전용이다.

## 캐릭터 표시

| 안전자금 방향 | 관리 방식 | 스킨 |
|---|---|---|
| 확보 우선 | 전체 | 든든 저금통 |
| 균형 | 전체 | 물결 저금통 |
| 활용 우선 | 가이드·적극 관리 | 무럭 저금통 |
| 활용 우선 | 자율 | 반짝 저금통 |
| 판단 보류 | 전체 | 기본 저금통 |

스킨은 기존 V2 페르소나를 결정론으로 번역한 표시값이다. 점수·보상·상품 조건·실행 승인에는
영향이 없다. 모든 스킨은 긍정적 성장 표현만 사용하고 낙인이나 우열을 만들지 않는다.

## 결선 데모 흐름

1. 홈에서 `320점 · 잠정 · 검증자료 준비 중`과 소득리듬 맵을 함께 보여준다.
2. 홈택스를 연결한다.
3. `360점 · 확정 · 검증자료로 심사 연결`로 바뀌고 초록 검증 발판이 한 칸 열린다.
4. 입금 배분을 승인한다.
5. 점수와 검증 단계는 그대로 유지되고, 파란 소득리듬 발판만 한 칸 열린다.
6. 상품 화면에서는 자체 한도를 보여주지 않고, 연결된 자료와 함께 실제 심사를 확인하도록 보낸다.
